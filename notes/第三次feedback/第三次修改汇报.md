## 一、反馈项闭环(13/13)

### 三个核心痛点(整体大改,方案一)

| 痛点 | 解法 | 状态 |
|---|---|---|
| 「频道」一词多义 | 三层正名:**我的账号**(资产)/ **对标账号**(一等实体,独立建库)/ **项目**(执行单元,Muse·Poet 归入项目内) | ✅ |
| 时长逻辑倒置(写稿才限时长) | 时长前置为**项目属性**,选题分析阶段即按目标时长工作 | ✅ |
| SOP / 圣经无法全局沉淀复用 | **SOP 库**全局化:任何账号/对标拆出的 SOP,任意项目一键选用;圣经多版本 + 项目级选用 | ✅ |

| 反馈 | 结果 |

| Clerk 术语翻译腔(社会仪式 CTA 等) | 全部替换为行业通用说法 |

| 左侧导航按 agent 排列 | 重构为「全局工具 / 我的账号」两组,执行入口收进项目 |

## 二、本轮新增能力(超出反馈范围)

**1. 拆对标直通链路**:Clerk 可直接拆解对标账号(不再需要伪装成自己的账号),产出 SOP 自动进库、带来源标识;历史伪账号一键「转为对标」。实测:拆解 Fireship → 它的 SOP 驱动徕卡频道写稿,风格迁移成立。

**2. 内容可信度三层防线**:反编造校对(删无依据的价格/参数/引语)→ 参考资料逐条事实核查(✓已核实 / ⚠存疑+建议值,带来源引用)→ 无字幕频道的 SOP 自动披露素材边界、禁止伪造时间码。

**3. 中文转写打通**:小红书笔记转写 1/15 → **15/15**;中文长视频(YouTube)从全军覆没修复为全量成功(分片转写),无字幕中文频道的分析质量从"只看标题封面"恢复到全文级。

**4. 运行进度与剩余时间**:所有长任务实时进度条 + 基于历史的预计剩余时间;逐视频进度轨道;全局运行指示器。

**5. 工作台与导航重设计**:工作台改为调度台(每个账号一条「下一步」行动入口 + 正在进行 + 最近动态);导航精简到 4 项。
