# Round 4 修改汇报（2026-06-26 至 07-01）

## 网站结构重设计

- 导航重构：Clerk 置顶（对标账号 / SOP 库 两个 tab）；「我的账号」是可展开的主从视图；删除工作台.
- 账号页以「频道圣经 + 项目」为主，复盘降为一行，「转为对标」入口挪到这里。
- 项目主页精简成 Muse / Poet 入口 + 当前用的 SOP 和对标 + 最近脚本。
- 频道圣经收敛到账号维度，不再散落在 Poet 和 Muse 里。
- 创建账号 / 项目改成弹窗，不再跳页；账号 URL 可选（圣经取代了原来的描述字段）。
- 删除类操作统一成确认弹窗。

## 多项目

- 一个账号下可以建多个项目，项目之间数据隔离（选题、SOP、脚本各归各）。
- Clerk 拆出来的 SOP 可以「在项目中选用」，一份 SOP 多个项目复用；SOP 库里能看到每份被哪些项目在用。

## Clerk（看对标 / 拆解）

- 新增单篇视频拆解：对某一条已分析的视频单独出一份深度 SOP，不影响频道整体 SOP。图文帖没有语音，不支持单篇拆解（会提示）。
- 频道 SOP 分层生成，长视频和多视频不再丢内容。英文视频的中文 SOP 会保留英文原话，不会翻译后当成原话引用。
- 对标账号、自有账号可手动刷新数据。
- 支持直接粘贴小红书 / YouTube 的分享卡链接解析, 改好了。

## Muse（选题）

- 选题会贴合该账号圣经的定位，不再偏离账号方向。
- 分析跟随对标视频本身的语言，生成的选题用你的目标语言（比如英文对标也能出中文选题）。
- 采用 → 直接交给 Poet 写稿（带着原视频转写和核查过的事实）；忽略 → 软删除。选题分「待处理 / 已采用 / 已忽略」三组，两个操作都能撤销。
- 开始监控时可以临时拉入还没绑定的对标账号。

## Poet（写稿）

- 时长控制：按设定时长出稿。中间时长（如 3 分钟）较准；30 秒以内的超短稿和 10 分钟以上的长稿会略偏长，时长按近似看。
- 长稿结构修正：高潮在行动号召之前，结尾不烂尾。
- 写稿保留素材里的不确定表述，「待核实」的事不会被写成绝对数字或结论。

## 界面与数据

- 全站 markdown 正常渲染。
- SOP 库支持删除单条，或删除某个账号的整套 SOP。
- 小红书数据接口迁移到新版（旧接口即将下线），视频和图文抓取正常。小红书视频走音频转写，图文走图片视觉识别。这个是月初收到Tikhub的邮件, 现在api已经都更新了.
